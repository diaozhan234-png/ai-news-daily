OpenAI updates its Agents SDK to help enterprises build safer, more capable agents
Agentic AI is the tech industry’s newest success story, and companies like OpenAI and Anthropic are racing to give enterprises the tools they need to create these automated little helpers. To that end, OpenAI has now updated its agents software development toolkit (SDK) , introducing a number of new features designed to help businesses create their own agents that run on the backs of OpenAI’s models. The SDK’s new capabilities include a sandboxing ability, which allows the agents to operate in controlled computer environments. This is important because running agents in a totally unsupervised fashion can be risky due to their occasionally unpredictable nature. With the sandbox integration, agents can work in a siloed capacity within a particular workspace, accessing files and code only for particular operations, while otherwise protecting the system’s overall integrity. Relatedly, the new version of the SDK also provides developers with an in-distribution harness for frontier models that will allow those agents to work with files and approved tools within a workspace, the company said. (In agent development, the “harness” is a term that refers to the other components of an agent besides the model that it’s running on. An in-distribution harness often allows companies to both deploy and test the agents running on frontier models, which are considered to be the most advanced, general-purpose models available.) “This launch, at its core, is about taking our existing Agents SDK and making it so it’s compatible with all of these sandbox providers,” Karan Sharma, who works on OpenAI’s product team, told TechCrunch. The hope is that this, paired with the new harness capabilities, will allow users “to go build these long-horizon agents using our harness and with whatever infrastructure they have,” he said. Such “long-horizon” tasks are generally considered to be more complex and multi-step work. OpenAI said it will continue to expand the Agents SDK over time, but initially, the new harness and sandbox capabilities are launching first in Python, with TypeScript support planned for a later release. The company said it’s also working to bring more agent capabilities, like code mode and subagents, to both Python and TypeScript. The new Agents SDK capabilities are being offered to all customers via the API, and will use standard pricing.

OpenAI更新其代理SDK，以帮助企业构建更安全、更强大的代理
Agent AI是科技行业最新的成功案例，OpenAI和Anthropic等公司正在竞相为企业提供创建这些自动化小助手所需的工具。为此，OpenAI现在更新了其代理软件开发工具包（SDK），引入了许多新功能，旨在帮助企业创建自己的代理，这些代理在OpenAI模型的支持下运行。SDK的新功能包括沙盒功能，允许代理在受控的计算机环境中运行。这很重要，因为以完全无监督的方式运行代理可能会有风险，因为它们偶尔是不可预测的。通过沙箱集成，代理可以在特定工作空间内以孤立的方式工作，仅访问特定操作的文件和代码，同时保护系统的整体完整性。该公司表示，与此相关的是，新版SDK还为开发人员提供了一个用于前沿模型的分发内工具，允许这些代理在工作空间内使用文件和批准的工具。（在代理开发中，“工具”是指代理运行的模型之外的其他组件。分发中的工具通常允许公司部署和测试在前沿模型上运行的代理，前沿模型被认为是最先进的通用模型。）OpenAI产品团队的Karan Sharma告诉TechCrunch：“这次发布的核心是采用我们现有的代理SDK，使其与所有这些沙盒提供商兼容。”。他说，希望这一点，再加上新的线束功能，将允许用户“使用我们的线束和他们拥有的任何基础设施来构建这些长期代理”。这种“长期”任务通常被认为是更复杂和多步骤的工作。OpenAI表示，随着时间的推移，它将继续扩展代理SDK，但最初，新的工具和沙盒功能将首先在Python中推出，计划在稍后的版本中支持TypeScript。该公司表示，它还在努力为Python和TypeScript带来更多的代理功能，如代码模式和子代理。新的代理SDK功能将通过API提供给所有客户，并将使用标准定价。
